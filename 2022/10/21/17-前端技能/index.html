Docker 部署

Docker 是一个开源的应用容器引擎，用于创建包含项目镜像的可移植容器并发布到 Linux 机器而实现虚拟化，核心概念有 Image 镜像 (包含应用运行代码及其依赖和配置文件的可执行包)、Container 容器 (镜像的运行时实例，相互之间没有任何接口的沙箱机制)、Repository 仓库 (存储可复制的镜像)。Docker 被请求运行容器时会在计算机上创建独立进程的隔离环境和应用及其依赖的镜像，然后通过执行预定命令来运行应用。以下为常见的部署方式：

  • 物理服务器直接部署：缺点是部署缓慢 (安装操作系统、应用程序、依赖软件等)、部署和迁移成本高 (简单应用也要一台服务器而且迁移时需要全部重新部署)、不能充分利用资源。
  • 通过虚拟机部署：虚拟机指在物理计算机通过软件层分出部分硬件资源而模拟具有完整功能但运行在独立环境的系统，一台服务器上模拟多台虚拟机就可以安装多个相互隔离的应用程序。优点有：可达到硬件资源的最大化利用、更容易扩展应用、虚拟出不同的物理资源而可以快速搭建云服务。缺点则是每台虚拟机都需要模拟整个操作系统而造成占用空间大和性能较差。
  • 通过轻量级容器技术 Docker 部署：容器指将操作系统层虚拟化的一个软件单元，通过底层 Linux 技术对进程进行封装为隔离于宿主机的独立进程。相比虚拟机在物理资源层面的隔离，Docker 在操作系统进程层面的隔离实现了小环境的轻量级虚拟。

Jenkins 工具

一个基于 Java 开发的持续构建工具平台，主要用于持续、自动的构建/测试软件和项目。它可以执行预定的设置和构建脚本，也可以集成 Git 代码库，实现自动触发和定时触发构建。

lerna 管理包

用来优化托管在 git/npm 上的多包代码库的工作流的一个管理工具，可以让你在主项目下管理多个子项目，从而解决了多个包互相依赖并且发布时需要手动维护多个包的问题。

Prettier 代码风格

GitHooks 提交规范

GitHooks 是指在 git 操作时触发运行 .git/hooks 目录下的钩子脚本，自定义钩子操作时如果直接修改 hooks 脚本并去掉 sample 后缀即可生效但是不能提交修改代码，所以一般通过引入工具 husky/yorkie 来配置。Commitlin 用来配置提交信息校验规则的工具。

三、GUI 桌面应用

基于显示器和一台处理器为用户提供功能并部署在计算机桌面的应用程序，常见的有 操作系统 (Windows/Mac/Linux)、文件管理器、数据库工具、音乐播放器、计算器等。传统开发方案有 VB 拖拽式开发、C++ duilib (微信和爱奇艺)、Java swing/javaFx (跨平台)、C++ MFC/Qt (开发 Window 应用)、Objective-c/swift-cocoa (开发 Mac 应用)。

跨平台框架

  • NW.js：基于 Chromium (渲染)、NodeJS (后端) 但通过合并事件循环源码实现通信而耦合性高，而且和 Electron 一样体积大和占用资源高(不推荐)，经典案例有微信/京东小程序 IDE。
  • Electron：基于 Chromium、NodeJS 通过独立线程通信，需要性能优化，案例有 VSCode。
  • Flutter：通过 Dart 语言实现基于 Skia 的高性能渲染，但需要原生开发。
  • Wails：基于 Go 语言实现高性能的轻量应用，可看作 Electron 替代品。
  • Tauri：通过 Rust 语言构建体积小、性能高、安全性强的二进制文件。

JS 数据结构

发表于 2019-05-20 更新于 2023-05-11 分类于 Javascript
本文字数： 18k 阅读时长 ≈ 16 分钟
栈、队列、链表、集合、字典、散列表、树、图
  • 逻辑结构：数据组织方式。
    • 集合结构：由一组无序、唯一、无关联的元素组成，主要操作有并集、交集、补集。
    • 线性结构：堆、栈、队列、一维数组的线性表，元素是有序的并有一对一的层次关系。
    • 树形结构：树，元素之间存在一对多的层次关系。
    • 图形结构：图，元素之间存在多对多的层次关系。
  • 物理结构：数据存储方式。
    • 顺序存储：元素连续放置而基于位置访问内存，可随机访问但大小固定和增删效率低。
    • 链式存储：元素不连续放置而基于指针的引用关系访问内存，大小动态扩展而插入删除效率高，但是访问链表元素时需要循环查找而不能随机访问。
    • 索引存储：内存整体无序但索引块有序，优化了顺序查找但需要额外空间存储索引表。
    • 散列存储：数据元素根据函数计算存储位置，基于数据本身查找而查找和存取效率高，但是存取随机而不利于顺序查找，存储位置可能相同而冲突。

一、栈

遵循先进后出原则的特殊线性表，基于数组或链表实现但只能在栈顶增删元素。

二、队列

遵循先进先出原则的特殊线性表，只能队尾添加和队首删除，类似排队人群。

三、链表

链接方式存储数据的线性表，是针对顺序存储结构线性表增删效率低问题的优化结果，每个数据单元就是一个包含节点信息和指向后一个节点引用地址的节点。

  • 单链表：单一的向下传递，每个节点只记录下一个节点的信息。
  • 静态链表：基于数组实现的链表，每个节点包含了数据和指向地址。
  • 循环链表：由于单链表到达尾部时回溯到头部很麻烦，所以首尾相连形成循环。
  • 双向链表：基于单链表增加前指针，每个节点都保存两个分别指向前后节点的地址。

五、字典

以键值对的形式存储不重复的数据，Object 对象就是字典在 Js 中的实现。

六、散列表

又称哈希表，是通过关键字将数据映射到一个内存位置而生成的一种便于搜索的数据结构。散列函数就是通过数字分析法、平方取中法、分段叠加法、除留余数法等实现的映射函数，其原则为：原则：函数本身便于计算、计算出来的地址分布均匀。注意键值不同的元素可能会映射到散列表的同一地址造成 hash 冲突，处理冲突的方式主要如下：

  • 再散列法：冲突时通过其它散列函数计算地址。
  • 建立公共溢出区：将所有和散列表冲突的元素都放入溢出表。
  • 拉链法/分离链接：将所有散列地址相同的记录都放入一个线性链表。
  • 开放定址法：冲突时寻找散列表中的空地址。线性探测 (顺序查看表中下一单元)、二次探测 (在表的左右进行跳跃式探测)、随机探测 (建立一个伪随机数发生器)。

七、树

相同类型元素的有限集合、一对多的非线性结构、分层数据的抽象模型。主要属性有：节点 (没有父节点的根节点、有子节点的分支节点、没有子节点的叶子节点)、子树 (节点组成的树)、层级 (根据节点级别分层)、深度 (到根节点的节点数量)、高度 (节点深度中的最大值)、权值。复杂的树状结构都需要转换为二叉树处理，常用类型如下：

  • 二叉树：每个节点最多含有两个子树，子节点之间有顺序关系。
    • 完全二叉树：最后一层右边不定，左边和其余层都是满的，比如堆。
    • 满二叉树：所有叶节点都在最底层的完全二叉树。
    • 排序二叉树：有序二叉树、二叉查找树、二叉搜索树。
    • 平衡二叉树：AVL树，任何节点的左右子树的高度差不超过 1。
  • 哈夫曼树：最优二叉树，带权路径最短的二叉树。
    • 路径长度：两个节点之间所经过的分支数目。
    • 树带权路径长度：所有叶子节点的带权路径长度之和。
    • 节点带权路径长度：节点到根节点的分支数目和该节点权值的乘积。
  • B 树：一种优化读写操作的二叉查找树，保持数据有序并有多余两个子树。

八、图

由一系列顶点和连接顶点的边组成的一种非线性数据结构，可以用来表示任何二元关系，采用哪种方式取决于图的类型和待解决的问题。

  • 邻接矩阵：通过一个二维数组表示顶点连接，但可能浪费内存。
  • 邻接表：由每个顶点的相邻顶点列表组成的一对多结构，比如数组、链表、散列表、字典。
  • 关联矩阵：矩阵的行列分别表示顶点和边并通过二维数组表示连接，常用于边的数量比顶点少时节省空间和内存。如果顶点 A 是边 E 的入射点，则 array[A][E] === 1，否则为 0。

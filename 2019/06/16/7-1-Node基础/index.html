Node.js 主要内容

发表于 2019-06-16 更新于 2023-05-16 分类于 JS 扩展
本文字数： 19k 阅读时长 ≈ 17 分钟
基础介绍、模块机制、应用场景

一、基础介绍

运行在服务端的 JS 环境，是基于 Chrome JavaScript V8 引擎建立的平台，用于开发和运行视频流网站、单页应用或在线聊天应用等io密集型网络应用。它封装了 V8 引擎并引入 CommonJS 规范而解决了 JS 比较慢和乱的问题，通过单线程和异步事件驱动机制实现了较强的并发处理能力，适用于高并发、I/O 密集、少量业务逻辑的场景。但是不适合处理密集计算和复杂逻辑，因为那可能会阻塞应用中运行的单线程。

  • 优点：轻量高效、事件驱动模型、异步非阻塞 I/O 模型、单进程单线程应用程序。
  • 不足：不适合处理复杂逻辑、不适合计算密集型应用、不适合单用户多任务的程序、代码某个环节报错则整个系统崩溃、只支持单核 CPU 而不能充分利用多核 CPU 服务器。
  • 前端应用：模块捆绑器(类似 Webpack/Vite 将代码片段捆绑为一个文件而方便运行)、代码润色器(类似 ESLint 检测和修改代码)、npm 包(帮助编程)、样式设计(编写组件库)。
  • 后端应用：数据库集成(支持 JS 进行数据库操作)、实时应用(事件驱动机制和非阻塞I/O模型实现密集通信)、网络和API调用(在线论坛等支持处理请求的同时继续其它工作)。
  • 应用案例：Uber(快速处理大量数据和修正故障)、Netflix(视频流媒体服务)、LinkedIn(提升性能和资源利用率)、贝宝(允许通过 JS 开发浏览器和服务器应用程序)。

运行机制

同步异步指操作任务是否按顺序执行，阻塞非阻塞指 IO 操作是否影响主线程执行其它代码。

  • 单线程：通过一个主线程处理所有的请求，注意异步操作是通过线程池来完成的。
  • 异步非阻塞：主线程接收到操作请求后就作为一个事件放入任务队列，然后继续接收其他请求，如果有耗时操作则交给不同的工作线程去执行，所以主线程速度快而且不会阻塞。
  • 事件驱动：主线程按顺序将所有操作都注册为一个事件并等待触发的方式。主线程首先执行完同步操作，然后通过事件循环机制查看任务队列中的事件并将异步事件交给工作线程。

功能架构

安装配置

本质是把 NodeJS 执行程序复制到一个目录 (建议不要安装在 C 盘而修改默认地址为 “D:\nodejs")，然后配置安装管理和环境变量以实现统一管理安装包和全局使用 node 命令。

二、模块机制

模块规范

  • 模块加载：CommonJS 规范的 require 加载、module.exports 导出。
    • 核心模块：本身已被编译好并存储在内存，只传入模块名称即可。
    • 第三方模块：存储于 node_modules，加载时从当前目录到根目录 node_modules 依次查找，和核心模块一样只需要传入模块名称而不要添加后缀和路径等。
    • 自定义模块：必须使用相对路径，引入文件夹则默认加载其 index.js。
  • 模块缓存
    • 模块会在第一次 require 时缓存，多次 require 不会导致模块代码被执行多次。
    • 浏览器 JS 只缓存文件，Node 模块缓存的是编译和执行之后的对象。
    • require.cache 对象代表 Node 模块缓存区，缓存模块以属性方式加入该对象，可以通过 require.cache[模块标识符] 访问具体的缓存模块，可以 delete 该缓存。
  • 模块变量
    • 全局对象：global (类似浏览器 window)、process (指向内置模块 process 允许开发者与当前进程互动)、console (指向内置模块 console 支持命令行环境的输入输出)。
    • 全局函数：require、setTimeout、setInterval、clearTimeout、clearInterval。
    • 全局变量：_filename、_dirname (指向当前运行脚本的文件名、所在目录)。
    • 模块变量：module (当前模块对象)、module.exports**：(模块的导出对象)、module.filename (模块解析后的文件名)、exports (指向 module.exports 的变量)。
  • 模块包规范
    • NodeJS 通过包括一个目录及其文件的包来统一管理一组具有相互依赖关系的模块。
    • 目录文件：package.json (描述文件)、bin (可执行的二进制文件)、lib (待加载的 js 文件)、doc (说明文档)、test (单元测试用例)、node_modules (本地依赖包)。

核心模块

Stream 流

一组有序的、有起止点的字节数据传输方式，本质是 buffer 对象的高级封装和 EventEmitter 类的实例，常用于文件上传、视频读取等异步操作。流类型可分为 Readable 可读、Writable 可写、Duplex 可读写、Transform 可转换，流模式则根据创建时传入参数 Object/Buffer 确定并且不可修改但是可拼接转换出一个新流，注意：Writable 流中写入 null 表示数据已写入完成而继续写入报错，Readable 流中读取到 null 表示数据已读取完毕。

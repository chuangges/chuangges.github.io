  • CPU：运算效率低的万能教授，处理各种不同数据和逻辑判断的中央处理器。
  • GPU：执行并发运算效率高的专业学生，专门执行复杂数学计算的图形处理单元。
  • OpenGL：最常用的跨平台图形库，用于渲染矢量图形的跨语言跨平台的应用接口。
  • OpenGL ES：OpenGL API 子集，是针对手机和游戏主机等嵌入式设备而设计的。
  • GLSL：OpenGL ES 着色语言，图形卡 GPU 上执行并实现视图转换等渲染管线功能。
  • WebGL：基于 OpenGL 面向 web 的图形标准，提供了一系列 JS API 可用于控制 GPU 渲染管线 (用于接收 CPU 传送的顶点和纹理等信息并处理为一张二维图像的显卡硬件)。
  • ThreeJS：一款封装 webGL 复杂接口的框架，简化工作有模型数据、各种矩阵、着色器等。
  • 坐标系：GPU WebGL 主要用于将顶点数据通过矩阵变换转换到裁剪空间坐标，然后基于空间坐标绘制像素点，无论画布多大都会将坐标限制到 (1, 1, 1)、(-1, -1, -1)。
  • 缓冲区对象：GPU 存储顶点和颜色等绘制模型的二进制数据。
  • 图元：几何顶点的组合 (点、线段和三角面)，一般是三角形。
  • 网格：使用顶点所绘的 2D、3D 模型，每个面被称为一个片元 (一般三角形)。
  • 片元和像素：光栅化的纹理映射后图元转为像素，片元指包括颜色纹理等的像素点像素点。
  • 着色器：GPU 顶点处理和片段处理等可编程部分，主要包括顶点和片元着色器。着色器程序使用 GLSL 语言编写并以字符串形式存于 JS 等待编译处理后传递给着色器执行。
  • 纹理图像：将一张真实图片贴到一个几何图像表面称为 纹理映射，映射的图像为 纹理图像，纹理图像的组成像素和坐标则分别被称为 纹素、纹理坐标 (可用于获取纹素颜色)。
  • 绘图：WebGL 只能绘制 点、线和三角形，将它们合成各种图形后绘制到三维空间。顶点、颜色、纹理等几何信息都存于缓冲区对象并被传递到着色器程序来创建图形对象。
  • 图像绘制流程：浏览器解析服务器硬盘的 WebGL 文件后由 CPU 将顶点数据和着色程序传递到 GPU 显存，CPU 调用着色程序初始化时创建的顶点/片段着色器和顶点数据传递到 GPU 处理器、GPU 通过矩阵变换和光栅化等操作将图像信息转化为像素并写入显卡等待读取。
  • GPU 流程：矩阵变换 – GPU 逐顶点执行顶点着色器而通过矩阵变换将 3D 顶点坐标转换为 2D 图元、图元装配 – 几何着色器通过新顶点裁剪图元并映射为像素、光栅化 – GPU 逐片元执行片元着色器生成像素着色后的片元、检测融合 – 检测片元的深度和透明度并混合。

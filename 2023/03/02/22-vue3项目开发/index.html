开发技术

一、开发技术

项目构建

  • Vite：开发环境时可快速冷启动，基于 esbuild 预构建依赖而无需分析模块依赖和编译代码，浏览器请求模块时才会进行按需动态编译，但是打包到生产环境时需要通过 rollup。
  • Webpack：启动后从入口开始逐步进行语法解析、依赖收集、代码转译、打包合并、代码优化等一系列操作，最终将高版本的、离散的源码编译打包成低版本、高兼容的代码。

版本优化

  • 源码体积的优化：重写了虚拟 dom。
  • 响应式系统的升级：Proxy + Reflect 取代 Object.definepeoperty() 而实现直接监听对象和数组而非属性，Proxy 返回的新对象可以直接操作而不需要每次都遍历对象属性。
  • 代码编译优化：Composition API 代替 Options API 而利于代码管理和类型推导、组件不需要根元素包裹而通过虚拟元素 fragment、标记静态节点而在 diff 时只需对比动态内容。
  • 生命周期变更：setup 创建实例前、onBeforeMount、onMounted 挂载前后、onBeforeUpdate、onUpdated 更新前后、onBeforeUnmount、onUnmounted 卸载前后。

$attrs

Vue2 可以通过 this.$attrs、this.$listeners 分别访问传递给组件的 props 未定义属性和事件监听器，Vue3 $attrs 包括了之前除外的属性 class、style 并合并移除了 $listeners。

provide/inject

实现树形组件的依赖注入功能，常用于层级嵌套较深的组件之间相互传递数据。

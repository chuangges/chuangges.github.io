四、Rxjs 异步

ReactiveX 编程理念的 Js 库。ReactiveX 来自微软，它是一种基于事件处理异步数据流的编程。简单来说，它将用户操作、网络响应、定时器、Worker 等一切数据都包装成流的形式，然后通过强大丰富的操作符对流进行处理，使你能以同步编程的方式处理异步数据，并组合不同的操作符来实现功能。

基本概念

  • Observable：可观察对象，一个可调用的未来值或事件的集合，用于存储流动的数据。
  • Observer：观察者，一个监听 Observable 流动结果的回调函数集合，用于作为参数传入订阅方法 subscribe 来获取数据流动结果。
  • Subscription：订阅，一个可执行的 Observable，主要用于取消订阅。
  • Operators：操作符，map、filter、concat、flatMap 等采用函数式编程风格的纯函数，用于处理数据。
  • Subject：主体，相当于 EventEmitter，用于将值或事件多路推送给多个 Observer。
  • Schedulers：集中式调度程序用来控制并发的调度器，用于将所有任务进行排序并控制执行时机 (计时器、动画等)。

API

数据流是一种可观察的序列，可以被订阅，也可以被用来做一些转换和组合。

  • 创建
    • 单值：of、empty、never
    • 多值：from
    • 定时：interval、timer
    • 事件：fromEvent
    • Promise：fromPromise
    • 自定义：create
  • 转换
    • 改变数据形态：map、mapTo、pluck
    • 过滤：filter、skip、first、last、take
    • 时间轴：delay、timeout、throttle、debounce、audit、buffer
    • 累加：reduce、scan
    • 异常处理：throw、catch、retry、finally
    • 条件执行：takeUntil、delayWhen、retryWhen、subscribeOn、ObserveOn
    • 转接：switch
  • 组合
    • merge：合并序列。
    • concat：保持原顺序连接两个数据流。
    • race：预设条件为其中一个数据流完成。
    • forkJoin：预设条件为所有数据流都完成。
    • zip：将各数据流最后一个值合并为对象。
    • combineLatest：各数据流最后一个值合并为数组。
  • 特殊 subject
    • AsyncSubject：结束时传送最后一个值。
    • ReplaySubject：新订阅时发送最后几个值。
    • BehaviorSubject：新订阅时发送当前的最新值。

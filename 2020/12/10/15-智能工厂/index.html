一、项目开发

智能工厂 DC 是基于 MEAN ( MongoDB + Express + Angular + Node.js) 框架扩展的全栈开发，主要包括：前端应用组件、Model 层、Service 层、后台的通用 CRUD、事物扩展、通用查询等。

开发准备

应用开发

查询优化

  • 尽量将能缩小范围的条件排在前面，减少数据的命中范围而提高查询效率。
  • 尽量少用 $or 和 $ne 来进行过滤，尽量避免使用关联表来进行查询。
  • Populate 关联表查询时尽量减少查询数据，要加上 select 选项。
  • 尽量减少查询数据库的次数，避免在 for 循环里查询数据多次数据库。
  • 查找、赋值用到枚举值类型时为方便维护使用枚举值，不要直接写数字。
  • 必须使用 try、catch 捕获代码错误，并在 catch 中写错误日志。

二、AngularJS

Google 开源的 JS 框架，AngularJS、Angular 分别指版本 angular1.x、angular2.0+。

核心概念

模块、组件、服务是核心特性，其它特性则基于它们衍生而出。比如：组件与服务之间有依赖注入特性、模块为组件和服务提供了编译的上下文和指令、管道等功能、视图更新依赖于双向绑定、视图的变换通过路由机制实现组件切换。

  • __模块__：将代码隔离为可复用的块，然后通过依赖注入将模块组织为一个完整的应用程序。
    • __JS 模块__：一个包含代码的独立文件，通过导入导出机制解决变量冲突和依赖关系。
    • __NgModule 模块__：一个组织代码的类，通过装饰器和元数据将组件、指令、服务等代码组织为功能单元，主要用来启动 angular 应用。
  • __组件__：通过页面指令等模板语法实现和视图层交互，将数据请求等任务委托给服务。
    • __数据绑定__：将模板和组件类的属性、方法建立映射关系。四种使用形式：插值表达式、属性绑定、事件绑定、双向绑定。
    • __指令__：向 HTML 元素添加自定义行为而实现动态视图。三种类型：结构指令（增删元素）、属性指令（修改属性）、组件（拥有模板的指令）。
    • __过滤器__：一种将输入值根据预定规则转换为页面显示值的转换函数。
  • __服务__：封装了日志服务等公共方法的独立模块，通过依赖注入提供给专注业务的组件。
  • __依赖注入__：在构造器创建组件时将依赖组件等以参数形式传入，组件内部直接使用依赖的功能，顶端依赖会首先构造则解决了空引用报错问题并降低了代码耦合度。

重要对象

  • __全局对象 angular__：全局可以调用
    • module：创建模块对象
    • copy、extend：拷贝数组、对象
    • forEach：遍历数组、对象
    • equals：两个对象是否相等
    • toJson、fromJson：序列化
    • bootstrap：手动启动 angular
    • lowercase、uppercase：字符串大小写转换
    • isString、isObject、isArray、equals：类型判断
  • __模块对象 angular.module()__：当前模块可以调用
    • config：配置模块，注入 provider/constance/内置服务
    • run：启动应用，初始化全局数据、支持注入所有服务
    • factory、service、provider、constant、value：服务
    • controller、filter、directive、animation：控制器、过滤器、指令、动画
  • __作用域对象 $rootScope $scope__：存储应用模型 ViewModel 的实例对象
    • $rootScope：ng-app 默认创建的根作用域对象
    • $scope：ng-controller 默认创建的模块作用域对象

指令

通过自定义 html 元素和属性实现扩展功能。注意：双括号是 ng-bind 的简写形式但存在数据闪烁问题，但单页面应用只有加载 index.html 时才存在该问题，所以 index.html 模板只使用 ng-bind、其它模板可以使用双括号、ng-bind。常用的内置指令如下：

  • __声明作用域__：ng-app、ng-controller。
  • __数据绑定__：双括号、ng-bind、ng-model、ng-init、ng-repeat。
  • __样式相关__：ng-class、ng-style、ng-if、ng-show、ng-hide、ng-switch。
  • __表单功能__：ng-value、ng-checked、ng-selected、ng-disabled、ng-readonly。
  • __事件绑定__：ng-click、ng-change、ng-dblclick、ng-mousedown、ng-mouseenter、ng-mouseleave、ng-mousemove、ng-mouseover、ng-mouseup、ng-submit。
  • __其它指令__：ng-src、ng-href、ng-include、ng-transclude。

过滤器

显示数据时对数据进行格式化或过滤，但不会真正改变原数据，常用的内置过滤器有：currency (货币处理)、date (日期格式化)、filter (子集)、limitTo (限制长度)、lowercase (小写)、uppercase (大写)、number (格式化数字)、orderBy (排序)。

服务

  • 内置服务：通过声明式依赖注入（定义形参）的方式引用。

    • __$http__：发送 http 请求
    • __$resource__：创建一个和服务器端数据交互的对象
    • __$window__：浏览器 window 元素的 jQuery 包装
    • __$document__：浏览器 document 元素的 jQuery 包装
    • __$rootScope、$rootElement__：根作用域、根元素的访问
    • __$cacheFactory__：提供键/值对放置到对象缓存
    • __$interval__：提供对 window.setInterval 访问
    • __$timeout__：提供对 window.setTimeout 访问
    • __$cookies__：提供对浏览器的 cookie 的读写访问
    • __$animate__：提供动画钩子来同时链接到基于 CSS、Js 的动画

  • 自定义服务

    • __factory__：返回基本类型或对象。
    • __service__：返回对象，默认构建了一个空白对象。
    • __value__：定义常量，值可以被修改，但不能被注入到 config。
    • __constant__：定义常量。不可改变和装饰，可以被注入到任何地方。
    • __provider__：可以注入到 config，必须使用 $get 返回一个对象。

三、环境部署

基础了解

部署方式

  • 物理服务器直接部署
    • 实现：首先准备一台物理服务器并安装操作系统，然后安装运行自己的应用程序。
    • 缺点：部署缓慢 (硬件服务器、操作系统、应用程序、依赖软件等)、成本高 (一个简单的应用也需要一台服务器)、简单浪费内存等硬件资源、迁移和扩展慢 (需要准备其他服务器重新部署)。
  • 通过虚拟机部署
    • 虚拟化技术：在一台物理计算机上，通过中间虚拟软件层 Hypervisor 隔离 CPU、内存等硬件资源，虚拟出多台虚拟服务器，这样可以在一台物理服务器上安装多个应用程序并且相互隔离。
    • 优点：可以把资源分配到不同的虚拟机而达到硬件资源的最大化利用、与直接在物理机上部署应用相比更容易扩展应用、虚拟出不同的物理资源而可以快速搭建云服务。
    • 缺点：在物理服务器创建一台虚拟机时，就需要虚拟出一套硬件并在上面运行完整的操作系统，每台虚拟机都占用许多的服务器资源。
  • 通过轻量级容器技术 Docker 部署
    • 实现：通过底层的 Linux 技术对 Linux 进程进行封装隔离，而被隔离的进程被称为容器，完全独立于宿主机的进程。Docker 是在操作系统进程层面的隔离，而虚拟机是在物理资源层面的隔离。
    • 优点：虚拟机太重，对宿主机资源消耗太大。Docker 作为可以轻松解决上述问题，让开发环境的安装、应用的部署变得非常简单。

SSH

  • 全称 secure shell protocol，一种专门为远程登录会话等网络服务提供的安全性协议，默认使用 22 号端口并采用密文的形式传输数据，可以有效防止远程管理过程中的信息泄露问题，相比 ftp、pop、telnet 等传统的网络服务程序具有更高的安全性。
  • 主要功能：提供类似 Telnet 的远程登录服务、提供类似 FTP 的传输数据服务。
  • 认证类型：通过用户名密码登陆的基于口令认证、通过密钥免密登陆的基于密钥认证。

Shell

本质是一个存在特定格式指令的文件，可通过脚本解释器解析并执行。用于读取读取并执行脚本。shell 既是应用程序又是一种脚本语言（应用程序解析脚本语言）。

四、数据库 MongoDB

一个基于分布式文件存储的数据库，是非关系数据库中功能最丰富、最像关系数据库的。它由 C++ 语言编写，支持基于 json 扩展的 bson 格式来描述无结构化的数据类型，可以存储比较复杂的数据类型，使用时可以搭配可视化管理工具 MongoChef。

基础了解

  • __核心概念__：数据库（仓库）、集合（数据）、文档（最小单位）。
  • __主要特点__：高性能、易部署、易使用，数据存储方便，但不支持事务和空间占用大。
  • __适用场景__：网站实时数据处理、基础缓存层（避免下层的数据源过载）、高伸缩性。
  • __不适用场景__：高度事务性的系统、传统的商业智能应用、复杂的跨文档/表级联查询。

数据类型

  • __ObjectID__：文档添加时自动生成的唯一 _id，类似主键的唯一 key。
  • __String__：UTF-8 字符串。UTF-8 是针对 Unicode 的一种可变长度字符编码。
  • __Boolean__：布尔值，取值 true、false。
  • __Integer__：整数，常用 Int32、Int64。
  • __Double__：浮点数，所有小数都是 Double。
  • __Arrays__：数组/列表，多个值存储到一个键。
  • __Object__：字典，形式为 key-value。
  • __Null__：空数据
  • __Timestamp__：时间戳
  • __Date__：日期

运算符

  • __比较运算符__：$gt 大于、$lt 小于、$eq 等于、$gte 大于等于、$lte 小于等于、$exists 是否存在、$in 包含、$ne 不等于、$nin 不包含。
  • __逻辑运算符__：$and 并且、$or 或者、$nor 对 or 的取反、$not 不存在、$mod 求模、$where 位置、$type 类型。
  • __排序__：$asc 升序、$desc 降序。
